## 2026-07-17 — 보안 감사(1차) + 발견 항목 전수 조치

**지시:** Cloudflare `security-audit` 스킬 적용해 보안 감사, 나온 문제 모두 해결, docs에 보안 폴더 만들어 로그 정리.

- 조치(커밋 `e4cf9ec`):

  - **M1** OAuth `state` 추가(서명쿠키 대조), **M2** 프로덕션 devCode 노출 차단·fail-closed, **M3** 신고글 삭제로 관리자 삭제 차단.

**운영자 조치 필요:** Render에 `ADMIN_USERNAMES`+`ADMIN_PASSWORD`(라이브 관리자 비번 교정), `RESEND_API_KEY` 설정 + 데모 계정 정리. 상세 [../security/remediation-log.md](../security/remediation-log.md).

## 2026-07-17 — 카카오 소셜 로그인

**지시:** 카카오톡/구글 등 소셜 로그인 추가. (카카오 먼저 — 사용자가 카카오 개발자 콘솔에서 앱 등록·Redirect URI 등록·REST API 키 발급.)

**한 일 (카카오):**
- `User`에 `provider`·`providerId`(+`@@unique`) 추가 — 소셜 계정 식별·중복 방지.
- [oauth_service.ts](../server/src/service/oauth_service.ts): authorize URL 생성 → code 토큰 교환 → 프로필 조회 → **계정 연결/생성**. 정책: ①이미 연결된 소셜계정이면 로그인, ②같은 이메일 계정 있으면 연결, ③없으면 신규 생성(username `kakao_<id>`, 닉네임은 카카오 닉네임 정리·중복 회피, 비밀번호는 사용 불가 랜덤 해시, `agreedAt` 기록).
- 라우트: `GET /auth/oauth/kakao`(동의화면 리다이렉트)·`GET /auth/oauth/kakao/callback`(로그인→세션 쿠키→프론트 복귀). Redirect URI는 요청 호스트로 구성(로컬/배포 동일 코드).
- 프론트 로그인 페이지에 **‘카카오로 로그인’ 버튼** + 콜백 에러(`?error=`) 안내. 정지 계정은 차단.
- 환경변수 `KAKAO_REST_API_KEY`(+선택 `KAKAO_CLIENT_SECRET`) — render.yaml에 선언(sync:false), 미설정 시 카카오 로그인 비활성.
- 타입체크(프론트·백) 통과, authorize 302 리다이렉트(client_id·redirect_uri 정확) 스모크 확인.

**남음:** 구글 로그인(같은 패턴), 첫 소셜 로그인 시 명시적 약관/닉네임 확인 화면(현재는 자동 가입+동의 기록).

**문서:** [logic/backend-requests.md](logic/backend-requests.md) 소셜 로그인 항목 추가.

## 2026-07-17 — 닉네임 + 이메일 인증 회원가입 (핸드폰·개인정보 동의 포함)

**지시:** ①닉네임 필드 신설(활동 표시명)·중복 확인, ②이메일 인증 후 가입 완료 + 한 이메일당 한 계정, ③비밀번호 보안 제약(앞 커밋 반영), ④회원가입 시 핸드폰 번호 + 개인정보 수집 동의. (발송은 **Resend** 외부 연동, 닉네임=활동 표시명 신설로 결정.)

**한 일 (백엔드):**
- `User`에 `email?`·`nickname?`(각각 unique)·`phone?`·`agreedAt?` 추가. 인증 대기 테이블 `EmailVerification`(가입 확정 전 임시 보관, 성공 시 User로 승격).
- **2단계 가입**: `POST /auth/register/start`(검증→인증코드 메일 발송)·`POST /auth/register/verify`(코드 확인→가입 확정+로그인). 실시간 중복확인 `GET /auth/check/nickname|email`.
- 검증: 이메일/닉네임 형식·중복, 닉네임 2~16자, **휴대폰 형식**(010-…), **개인정보 동의 필수**, 비밀번호 8자+영문·숫자. 가입 확정 시 `agreedAt` 기록.
- **이메일 발송**([email_service.ts](../server/src/service/email_service.ts)): Resend HTTP API. `RESEND_API_KEY` 없으면 콘솔에 코드 출력 + 응답에 devCode(테스트 모드). render.yaml에 `RESEND_API_KEY`·`RESEND_FROM`(sync:false) 선언.
- 프로필 수정(`PATCH /auth/me`)에 닉네임 변경(중복 확인) 추가. 공유 user select에 nickname 포함.

**한 일 (프론트):**
- 회원가입을 **2단계 화면**으로 재작성([register_page.tsx](../frontend/src/pages/register_page.tsx)): 정보 입력(이메일·닉네임 **실시간 중복확인**·핸드폰·동의 체크박스) → 인증코드 입력. 테스트 모드에선 코드 화면 표시.
- **닉네임 표시 전환**: `displayName(u)=nickname||name` 헬퍼([display.ts](../frontend/src/shared/lib/display.ts))로 홈 카드·서평/토론 상세·댓글·프로필·기록·마이페이지·네비 표시명 교체(관리자 대시보드는 실명 유지). 설정에 닉네임 편집 추가.
- 타입체크(프론트·백) 통과, 가입 start→verify E2E + 동의/폰 검증 스모크 확인.

**보류(결정/자격증명 필요):** **소셜 로그인**(카카오·구글 OAuth) — 앱 등록·client id/secret·리다이렉트·계정 연동 정책 필요. **모바일 앱(Android/iOS)** — 아키텍처 방향(웹뷰 래핑 vs React Native)·토큰 인증 전환 등 별도 논의.

**문서:** [logic/backend-requests.md](logic/backend-requests.md)(인증 2단계·닉네임·핸드폰·동의·Resend), [design/ui-requests.md](design/ui-requests.md)(2단계 가입·닉네임 표시) 갱신.

## 2026-07-17 — 다수 UX/버그/관리 기능 일괄 개선

**지시(한 세션에 연속 지시):** ①신규 유저 서재에 이전 유저 책이 남는 문제, ②서평은 서재에 담은 뒤에만 쓸 수 있게 안내, ③빈 서평(cf_bob/cf_alice)·"들어가면 다른 서평" 문제, ④"요즘 많이 사는 책" 담기 버튼 가독성·위치 통일, ⑤토론/서평 작성자 클릭 시 프로필로, ⑥관리자 회원 관리(권한·정지·삭제), ⑦토론 상세 책소개/본문 구별 + footer 뜸, ⑧메인 카드에서 작성자·책 개별 클릭 이동, ⑨과하게 짧은 서평·토론 방지, ⑩푸터 피드백/버그 신고 + 관리자 조회, ⑪이용약관 페이지, ⑫비밀번호 보안 제약.

**한 일:**

- **① stale 캐시(버그):** 원인은 백엔드 시드가 아니라 프론트 SWR 캐시. `/progress/me`·`/books/interests/me` 키가 고정 문자열인데 로그인/가입/로그아웃 시 캐시를 안 비워 이전 사용자 데이터가 잠깐 노출. [auth_context.tsx](../frontend/src/shared/lib/auth_context.tsx)에서 로그인·가입·로그아웃마다 `mutate(()=>true, undefined, {revalidate:false})`로 전역 캐시 클리어.
- **③ bookSeq 충돌(버그):** [progress_service.ts](../server/src/service/progress_service.ts) `saveProgress`가 순번을 `count+1`로 매겨, 서평 삭제 후 새 서평이 기존과 같은 `bookSeq`를 가져 `/books/:id/reviews/:seq`에서 다른 서평이 뜸. `maxBookSeqByBook`(repo 신규) 기반 `max+1`로 변경 + `findProgressByBookSeq`에 `createdAt asc` 고정. (cf_bob/cf_alice는 배포 DB의 테스트 계정 잔존 데이터 — ⑥ 회원 삭제로 정리 가능.)
- **②·⑨ 검증:** 서평은 **내 서재(관심) 책만** 선택 가능([quick_actions.tsx](../frontend/src/widgets/quick_actions.tsx)), 서재 비면 안내문. 과하게 짧은 글 방지 — 서평 note는 적었다면 10자↑(빈 값=쪽수 기록 허용), 토론 제목 2자↑·내용 10자↑. 백엔드([progress_service.ts]·[discussion_service.ts]) + 프론트 `minLength`.
- **④ 버튼 통일:** 담기/서평 버튼에 `card-action full` — 카드 하단 고정·폭 통일, 외부추천 담기 버튼을 ghost→채운 버튼으로 가독성↑([index.css] `.card-action`, [book_card.tsx], [home_page.tsx]).
- **⑤·⑧ 작성자/책 링크:** 홈 서평·토론 카드를 단일 Link→개별 링크 구조로(표지·글→상세, 책→책상세, 작성자→프로필). 서평 댓글 작성자도 프로필 링크. (`/users/:id`·`UserProfilePage`는 기존 존재.)
- **⑥ 관리자 회원 관리:** `User.suspended` 추가. `GET /admin/members`, `PATCH .../admin`, `PATCH .../suspend`, `DELETE .../members/:id`. 정지 계정은 로그인·`requireAuth`에서 차단(미들웨어 async화). 본인·다른 관리자 정지/삭제 방지 가드. 관리자 페이지에 회원 목록 UI.
- **⑦ 토론 상세:** 본문을 라벨(‘토론 내용’) 붙은 좌측 강조 카드로 분리, 책 줄에 📖 링크. **스티키 푸터**(`#root` flex column + `.container` flex:1)로 footer 뜸 해결.
- **⑩ 피드백:** `Feedback` 모델 + `POST /feedback`(비로그인 가능·레이트리밋) + 관리자 `GET/PATCH/DELETE /admin/feedback`. 푸터에 ‘💬 의견·버그 신고’ 위젯([feedback_widget.tsx]), 관리자 페이지에 목록·처리·삭제.
- **⑪ 이용약관:** [terms_page.tsx](../frontend/src/pages/terms_page.tsx) 정적 페이지 + `/terms` 라우트 + 푸터 링크.
- **⑫ 비밀번호 제약:** `validatePassword`(8자↑ + 영문·숫자) — 회원가입·비밀번호 변경에 적용, 가입 폼에 안내.

- **세션 서명 쿠키**: `cookieParser(SESSION_SECRET)` + `res.cookie(..., {signed:true})` + `req.signedCookies` → 클라이언트 쿠키 위조 차단(가장 치명적 결함 해결). `SESSION_SECRET` env 필요(미설정 prod는 부팅마다 임시 시크릿+경고).
- **관리자 판정을 서버 통제 컬럼으로**: `User.isAdmin` 추가, 유저명 매칭(`admin.ts`) 폐기·삭제. 가입으로 관리자 위장 불가.
- **신고 남용 방지**: `Report @@unique([reporterId,targetType,targetId])` + 중복 신고 멱등 처리.
- **인증 레이트리밋**: `express-rate-limit`(IP 15분 20회)로 login/register/change-password.
- **CSRF 간이 방어**: 상태변경 요청에 커스텀 헤더(`X-Requested-With`) 요구(프론트 client.ts가 부착).
- **기타**: bcrypt cost 8→12, 로그인 타이밍 균일화(더미 해시), 자기 서평 좋아요 차단, 별점 정수 검증(NaN 방지), import cover URL http(s) 검증.
- 스키마 `db push`(--accept-data-loss) + 재시드. **curl로 로그인·위조차단(401)·관리자(200/403)·CSRF(403)·신고중복(멱등) 검증 완료.**

**상세 리포트:** 스크래치패드에만 보관(취약점 상세라 공개 저장소 미포함).
