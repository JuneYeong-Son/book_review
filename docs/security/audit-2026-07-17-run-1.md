## 상세

### C1 — 시드 공개 관리자 (CRITICAL)
- **문제:** `seed.ts`가 `reader`를 비번 `password`인 관리자로 생성(최초 배포 시 실행), `bootstrap.ts`가 기본값 `reader`를 매 부팅 관리자 재승격. → 누구나 `reader`/`password`로 관리자 로그인.
- **조치:** 프로덕션 시드는 데모 계정 **랜덤 비번**(로그인 불가), 관리자 비번은 `SEED_ADMIN_PASSWORD`/랜덤. bootstrap 기본 `reader` **제거**(ADMIN_USERNAMES 명시 필수) + `ADMIN_PASSWORD` 설정 시 부팅 시 관리자 비번 재설정.
- **⚠️ 운영자 조치(라이브 기존 데이터):** Render에 `ADMIN_USERNAMES`(본인 계정)와 `ADMIN_PASSWORD`(강한 값) 설정 후 재배포 → 라이브 관리자 비번 교정. 데모 계정(bookworm/soyul/cheol/essay, 기존 `password`)은 관리자 페이지에서 삭제/비번변경 권장.

### H1 — OAuth 이메일 미검증 연결 (HIGH)
- **문제:** 소셜 이메일이 기존 계정과 같으면 제공자 검증 여부 확인 없이 연결·로그인 → 미검증 이메일로 피해자 계정 탈취 가능.
- **조치:** `is_email_verified`(kakao)/`verified_email`(google)가 true일 때만 이메일을 연결·생성에 사용. 미검증이면 이메일 무시(신규 소셜 계정만 생성).

### M1 — OAuth state 부재 (MEDIUM)
- **문제:** authorize에 `state` 없음, 콜백은 쿠키 설정 GET → 로그인 CSRF/세션 고정.
- **조치:** 시작 시 난수 `state`를 서명 쿠키(`oauth_state`, 10분)로 저장, 콜백에서 대조·불일치 시 거부.

### M2 — 이메일 인증 우회 + 코드 노출 (MEDIUM)
- **문제:** dev 모드가 `NODE_ENV`가 아닌 키 유무로 결정 → 프로덕션에서 키 없으면 `register/start`가 인증코드를 응답에 노출 → 남의 이메일로 가입.
- **조치:** 프로덕션 + 키 없음 → 발송 실패(가입 차단), `devCode`는 비프로덕션에서만 반환.

### M3 — 신고글 삭제로 관리자 삭제 (MEDIUM)
- **문제:** `deleteReportedPost`의 user 분기가 `removeMember`의 "다른 관리자 삭제 금지" 가드를 건너뜀.
- **조치:** 대상이 관리자면 거부.
