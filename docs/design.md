# 책갈피 디자인 시스템 (design.md)

이 문서는 프론트엔드의 시각 언어를 규정한다. 새 화면·컴포넌트를 만들 때는 여기 정의된
토큰과 규칙을 **그대로 사용**한다. 구현 토큰은 [frontend/src/app/styles/index.css](../frontend/src/app/styles/index.css)의 `:root` 변수와 1:1로 맞춘다.

---

## 1. Overview — 디자인 철학

**"종이 위에 쓴 느낌(Ink on Paper)".** 화면을 "빛나는 스크린"이 아니라 **오래된 종이 한 장**처럼 다룬다. 그 위에 잉크로 눌러쓴 것처럼 차분하고, 읽기에 집중된 분위기를 지향한다.

- **전체가 종이톤 + 실제 종이 질감.** 순백(`#fff`)이 아닌 따뜻한 크림/양피지색 바탕에, 실제 종이 사진 텍스처를 깐다(§2).
- **글씨는 을유1945.** 제목·본문 전부 을유1945(옛 활자/필기 느낌)로, 손으로 눌러쓴 인상을 화면 전체에 준다.
- **요소는 종이에 인쇄된 듯.** 카드·패널은 반투명이라 뒤 종이 결이 비친다. 요소가 종이 위에 둥둥 뜨지 않고 **얹혀 붙어** 보이게 한다. 구분은 되되 경계는 도드라지지 않게.
- **색은 절제.** 세피아 잉크를 기본으로, 포인트는 **브론즈/오커** 하나. 장식·그림자는 최소로, 텍스트가 먼저 읽히게 한다.
- **단일 라이트 테마.** 다크 모드는 지원하지 않는다(`color-scheme: light`). 단, 푸터 등 "페이지를 닫는 면"은 에스프레소 다크를 예외로 쓴다.

원칙: **종이답게, 조용하게, 콘텐츠 먼저.** 애매하면 요소를 빼는 쪽을 택한다.

> 이전의 "고요한 서재(숲색 accent·명조/고딕)" 컨셉은 폐기됨. 현재 방향은 위 "종이 위에 쓴 느낌".

## 2. 종이 질감 · 필기 폰트 · 반투명 (핵심 · 필수 유지)

> 사용자가 지시한 핵심 방향. 색·레이아웃을 개편하더라도 아래는 **깨지 말고 유지**한다.

### 2-1. 배경 — 실제 종이 질감
- 페이지 바탕(`body`)에 종이 사진 텍스처([assets/textures/paper-bg.avif](../frontend/src/assets/textures/paper-bg.avif))를 `background-size: cover`로 깐다.
- 그 위에 **반투명 크림 베일**(`--bg` 색을 `rgba(…, ~0.32)`로)을 덮어 크림톤과 가독성을 지킨다. **질감 강도는 이 베일 알파값 하나로 조절**(낮출수록 결이 선명).
- **배경은 콘텐츠와 함께 스크롤한다.** `background-attachment: fixed` **금지** — 종이가 뷰포트에 고정돼 내용만 그 위로 지나가는 패럴랙스는 쓰지 않는다. 종이와 콘텐츠가 **한 장처럼 같이** 움직인다.
- 이미지 파일이 저화질이면 `cover`로 늘렸을 때 깨진다. 고화질 원본을 쓴다(현재 AVIF).

### 2-2. 글씨 — 을유1945
- 제목·본문·UI **전부** 을유1945(`--font-display` = `--font-body`). 특정 요소만 다른 폰트로 바꾸지 않는다.
- 자체 호스팅([assets/fonts/eulyoo/](../frontend/src/assets/fonts/eulyoo/)), `@font-face`는 [fonts.css](../frontend/src/app/styles/fonts.css), `font-display: swap`.
- 폴백은 나눔명조 → Batang → serif(종이톤과 통일감). 붓/펜 스크립트 등 다른 필기체로 대체하지 않는다.

### 2-3. 요소 — 반투명("종이에 인쇄된" 느낌)
- 카드·패널 면(`--surface`)은 **반투명**(alpha ~0.6)으로 두어 뒤 종이 결이 비치게 한다.
- **구분은 되되 도드라지지 않게** — 테두리는 종이톤에 가까운 부드러운 hairline(`--border`), 그림자는 hover·메뉴·모달에만(§5). 대비를 세게 주지 않는다.
- **예외 — 오버레이는 불투명.** 위에 떠서 가독성이 중요한 **모달·드롭다운·내비게이션**은 반투명하면 뒤 내용이 비쳐 지저분하므로 **불투명 종이(`--surface-solid`)**를 쓴다.

## 3. Colors — 역할 기반 색 사용

색은 "예뻐서"가 아니라 **역할**로 쓴다. 아래 토큰 외의 임의 색상 값은 쓰지 않는다. 실제 코드값은 [index.css](../frontend/src/app/styles/index.css) `:root`가 단일 소스.

| 토큰 | 값 | 역할 |
|------|-----|------|
| `--bg` | `#f2ece0` | 페이지 바탕(종이/양피지). 가장 넓은 배경 |
| `--surface` | 반투명 크림 `rgba(235,227,210,0.6)` | 카드·패널 면. **반투명**이라 종이 결이 비침(§2-3) |
| `--surface-solid` | `#ece4d3` | 모달·드롭다운·내비 등 오버레이(불투명) 면 |
| `--surface-soft` | 반투명 `rgba(238,231,216,0.5)` | 옅은 구분 밴드 |
| `--ink` | `#272016` | 본문·제목 글자(따뜻한 세피아 잉크) |
| `--muted` | `#6f6551` | 보조 텍스트(메타·설명·placeholder) |
| `--accent` | `#a87132` | **주 강조**(브론즈/오커): 주요 버튼, 링크, 활성, 브랜드 |
| `--accent-active` | `#895d28` | accent press(눌림 시 어둡게) |
| `--accent-ink` | `#fbf7ee` | accent 배경 위 글자(순백 대신 크림) |
| `--accent-soft` | `#e4d9c1` | accent의 옅은 배경(태그, hover 면, 히어로) |
| `--gold` | `#c08a3a` | **장식 강조**(앤티크 골드): 별점, 글귀 강조선 |
| `--danger` | `#c0392b` | 파괴적/경고: 삭제, 에러, 좋아요(♥) |
| `--border` | `#e2d8c6` | 부드러운 hairline 경계선·구분선 |
| `--surface-dark` | `#211b12` | 에스프레소 다크 면(푸터 등, 반전 없음) |
| `--on-dark` | `#f2ece0` | 다크 면 위 글자 |
| `--on-dark-soft` | `#a99f89` | 다크 면 위 보조 글자 |

**규칙**
- 텍스트는 `--ink`(본문) / `--muted`(보조) 두 단계만. 저대비 남발 금지.
- 강조는 `--accent`(브론즈) 하나가 기본. `--gold`는 별점·글귀 같은 "장식적" 강조에만 제한.
- `--danger`는 되돌릴 수 없는 행동과 에러에만. 일반 강조에 빨강을 쓰지 않는다.
- 배경 위계는 `--bg`(바탕) → `--surface`(면) 2단계. 면 위에 또 면을 겹칠 땐 `--border`로 구분.
- 다크(`--surface-dark`)는 푸터처럼 "페이지를 닫는 면"에만. 본문 영역을 다크로 반전하지 않는다.

## 4. Layout — 간격 · 타이포그래피

### 간격
- 여백·패딩·gap은 일관된 스케일(4의 배수 권장: 4·8·12·16·20·24·32·48·64)을 쓴다. 형제 요소 배치는 **flex/grid + `gap`**으로 하고, 개별 `margin`으로 간격을 만들지 않는다(상쇄·중복 방지).
- 콘텐츠 최대 폭: **960px**(`.container`), 좌우 패딩 24px, 하단 64px.
- 읽기 본문(서평 등)은 한 줄 **약 42em**(≈65자)로 제한해 가독성 유지.
- 넓은 콘텐츠(표·코드)는 자체 컨테이너에 `overflow-x: auto`. 페이지 본문은 가로 스크롤 금지.

### 타이포그래피

| 역할 | 폰트 | 크기 | 비고 |
|------|------|------|------|
| Display / 히어로 | `--font-display`(을유1945) | `clamp(1.6rem, 4vw, 2.5rem)` | `text-wrap: balance` |
| 섹션 제목 h2 | `--font-display` | ~1.2rem, 굵게 | 하단 `--border` 실선으로 구분 |
| 소제목 h3 | `--font-display` | ~1.05–1.15rem | |
| 본문 | `--font-body`(을유1945) | 1rem, line-height 1.5 | |
| 읽기 본문(서평) | `--font-body` | 1.12rem, line-height 1.85 | 여유 있는 행간 |
| 보조/메타 | `--font-body` | 0.82rem, `--muted` | `.small` |

- **제목·본문·UI 모두 을유1945**(`--font-display` = `--font-body`). 획이 굵어 제목은 자간 0 + 넉넉한 행간.
- 숫자가 줄맞춤되는 곳(별점·좋아요 수·페이지·통계)은 `font-variant-numeric: tabular-nums`.
- 말줄임은 `…`(ellipsis) 사용, `...` 금지. 인용부호는 곡선따옴표(" ").

## 5. Elevation — 요소가 떠오르는 방식

종이 위에 얇게 얹힌 느낌. 그림자는 **낮고 부드럽게**, 남용하지 않는다. 기본은 그림자 없이 **테두리(`--border`)로 구분**한다.

| 레벨 | 그림자 | 용도 |
|------|--------|------|
| **0 (flat)** | 없음 + `1px solid --border` | 카드·패널·입력의 기본 상태(반투명 `--surface`) |
| **1 (raise)** | `0 6px 18px rgba(0,0,0,0.08)` | 카드 hover, 살짝 떠오름 |
| **2 (menu)** | `0 8px 24px rgba(0,0,0,0.12)` | 드롭다운·팝오버·알림 메뉴(불투명 `--surface-solid`) |
| **3 (overlay)** | 반투명 배경 `rgba(0,0,0,0.45)` + 불투명 면 | 모달. 뒤 콘텐츠를 덮음 |

**규칙**
- 평상시엔 elevation 0(테두리)로 두고, **상호작용(hover)·일시적 표시(메뉴/모달)에만** 그림자를 준다.
- 그림자 색은 항상 중립 검정 알파. 컬러 그림자 금지(다크 면 위 소수 강조는 예외).
- 떠 있는 면(메뉴·모달)은 **불투명(`--surface-solid`)**으로 — 반투명이면 뒤가 비쳐 읽기 어렵다(§2-3).
- 모달은 배경 스크롤 잠금(`overscroll-behavior: contain`)과 함께 쓴다.

## 6. Shape — 모양(모서리·선)

살짝 각진 종이/책 느낌.

| 요소 | radius |
|------|--------|
| 버튼 | **8px** |
| 카드·패널·입력(input/textarea/select) | **12px** |
| 모달·히어로 | **16px** |
| 태그·뱃지·칩·알약형 버튼 | **999px** (완전 알약) |
| 표지 이미지 | 4–12px (거의 각짐) |
| 아바타·아이콘 버튼 | **50%** (원) |

**규칙**
- 컨테이너류는 위 값으로 통일한다(버튼 8 · 카드/입력 12 · 히어로/모달 16). 그 이상 큰 라운드는 쓰지 않는다.
- 알약형(999px)은 태그·칩·상태 뱃지 등 "라벨" 성격에만.
- 구분선은 `1px solid --border`. 서평 본문·댓글 경계 같은 강조 구분은 `2px solid`.

## 7. Components — 컴포넌트 규격

공통 요소는 아래 규격을 재사용한다. 새 변형이 필요하면 boolean prop을 늘리지 말고 **명시적 컴포넌트/클래스**로 분리한다.

### 버튼 (`.btn`)
- 기본: 배경 `--accent`, 글자 `--accent-ink`, radius 8px, 패딩 8/16, 굵기 500. 눌림(`:active`)만 `--accent-active`로 어둡게.
- `.btn.ghost`: 투명 배경 + `--border` 테두리 + `--accent` 글자. 보조 액션.
- `.btn.small`: 패딩 축소, 0.85rem. `.btn.full`: 폭 100%. `.btn.danger`: 배경 `--danger`.
- 아이콘 전용 버튼은 **반드시 `aria-label`**. 이모지는 `aria-hidden`.

### 카드
- 컨테이너: 반투명 `--surface` + `1px solid --border` + radius 12px, 내부 패딩 16.
- hover 시 elevation 1. 종이 결이 비치되(§2-3) 텍스트 가독성은 유지.
- **서평 카드(`.review-card`)**: 서평 내용이 주인공 — 표지는 작게, 본문(`.sq-review`)은 `--ink`로 최대 6줄, 하트·작성자는 하단 footer로 분리.

### 태그·뱃지 (`.tag`, `.page-badge`)
- 알약형(999px), 작은 글씨(0.72–0.78rem), `--accent-soft` 배경 + `--accent` 글자.

### 입력 (`input`, `textarea`, `select`)
- `#fff` 배경 + `--border` + radius 12px, 패딩 8/12.
- 포커스: `:focus-visible`에 `2px solid --accent-soft` outline + `--accent` 테두리.
- 라벨은 `<label>` 연결 또는 `aria-label`. 아이디/이메일류는 `spellCheck={false}`, 적절한 `autoComplete`.

### 별점 (`StarRatingDisplay` / `StarRatingInput`)
- 표시용과 입력용을 **분리**한다. 표시용은 `role="img"` + aria-label(비버튼), 입력용은 별마다 버튼.
- 채운 별 `--gold`, 빈 별 회색.

### 모달 (`Modal`)
- elevation 3(반투명 배경 + **불투명** `--surface-solid` 면), radius 16px.
- `role="dialog"` `aria-modal` `aria-labelledby`, **Escape 닫기**, 열릴 때 포커스 이동, 배경 스크롤 잠금.

### 내비게이션 / 드롭다운 / 탭 / 칩
- 내비·드롭다운은 **불투명 `--surface-solid`**(뒤 콘텐츠가 비치지 않게).
- 활성 상태는 `--accent`(글자/밑줄/배경)로 표시. 비활성은 `--muted`.
- 알림·아바타 등 토글 버튼은 `aria-haspopup`/`aria-expanded`.

### 푸터
- 에스프레소 다크(`--surface-dark`) 면 + `--on-dark` 글자. 페이지를 닫는 면(반전 없음).

### 접근성 공통
- 모든 상호작용 요소는 `:focus-visible` 포커스 링 필수.
- 비동기 에러/성공 메시지는 `role="alert"`/`role="status"`.
- 애니메이션·종이 그레인은 `prefers-reduced-motion: reduce`를 존중.
- 이미지는 `width`/`height` 지정(CLS 방지), 접힘 아래는 `loading="lazy"`.
