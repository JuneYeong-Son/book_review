# 책갈피 디자인 시스템 (design.md)

이 문서는 프론트엔드의 시각 언어를 규정한다. 새 화면·컴포넌트를 만들 때는 여기 정의된
토큰과 규칙을 **그대로 사용**한다. 구현 토큰은 [frontend/src/app/styles/index.css](../frontend/src/app/styles/index.css)의 `:root` 변수와 1:1로 맞춘다.

---

## 1. Overview — 디자인 철학

**"고요한 서재(Quiet Library)".** 오래된 종이 위에 잉크로 눌러쓴 책처럼, 차분하고 읽기에 집중된 분위기를 지향한다.

- **전체 분위기는 종이톤.** 순백(`#fff`)이 아닌 따뜻한 크림/종이색을 바탕으로 쓴다. 화면이 "빛나는 스크린"이 아니라 "펼친 책"처럼 느껴지게 한다.
- **주인공은 콘텐츠(서평·글귀).** 장식·색·그림자는 최소로, 텍스트가 먼저 읽히도록 여백과 위계를 쓴다.
- **포인트는 절제해서 한 곳에.** 숲색(bottle green)과 황동(brass)은 강조가 필요한 소수 요소에만.
- **제목·본문 모두 을유1945(필기 느낌의 옛 활자).** 문학적 인상을 화면 전체에 준다(제목=본문 동일 폰트).
- **단일 라이트 테마.** 다크 모드는 지원하지 않는다(`color-scheme: light`).

원칙: **조용하게, 종이답게, 콘텐츠 먼저.** 애매하면 요소를 빼는 쪽을 택한다.

## 1.5 종이 질감 · 필기 폰트 · 반투명 (필수 유지 방향)

> 사용자가 지시한 핵심 방향. 새 화면·리팩터·색 개편을 하더라도 아래는 **깨지 말고 유지**한다.

- **배경은 실제 종이 질감.** 페이지 바탕(`body`)에 종이 사진 텍스처([assets/textures/paper-bg.avif](../frontend/src/assets/textures/paper-bg.avif))를 `background-size: cover`로 깔고, 그 위에 **반투명 크림 베일**(`--bg` 색을 `rgba(…, ~0.32)`로)을 덮어 크림톤과 가독성을 지킨다. **질감 강도는 이 베일 알파값 하나로 조절**(낮출수록 결이 선명).
- **배경은 콘텐츠와 함께 스크롤.** `background-attachment: fixed` **금지** — 종이가 뷰포트에 고정돼 내용만 그 위로 지나가는 패럴랙스는 쓰지 않는다. 종이와 콘텐츠가 **한 장처럼 같이** 움직인다.
- **필기 폰트는 을유1945.** 제목·본문 전부 을유1945(`--font-display`=`--font-body`), 자체 호스팅([assets/fonts/eulyoo/](../frontend/src/assets/fonts/eulyoo/)), 폴백 나눔명조→Batang→serif. 붓/펜 스크립트 등 다른 필기체로 대체하지 않는다.
- **요소는 반투명 — "종이에 인쇄된" 느낌.** 카드·패널 면(`--surface`)은 반투명(alpha ~0.6)으로 두어 뒤 종이 결이 비치게 한다. 요소가 종이 위에 둥둥 뜬 게 아니라 종이에 얹혀 붙은 것처럼 보이게 한다.
- **구분은 되되, 도드라지지 않게.** 요소와 배경은 알아볼 수는 있으되 경계가 튀지 않게 한다 — 테두리는 종이톤에 가까운 **부드러운 hairline**(`--border`), 그림자는 hover·메뉴·모달에만(§4). 대비를 세게 주지 않는다.
- **예외 — 오버레이는 불투명.** 위에 떠서 가독성이 중요한 **모달·드롭다운·내비게이션**은 반투명하면 뒤 내용이 비쳐 지저분하므로 **불투명 종이(`--surface-solid`)**를 쓴다.

## 2. Colors — 역할 기반 색 사용

색은 "예뻐서"가 아니라 **역할**로 쓴다. 아래 토큰 외의 임의 색상 값은 쓰지 않는다.

| 토큰 | 값 | 역할 |
|------|-----|------|
| `--bg` | `#f2eee4` | 페이지 바탕(종이). 가장 넓은 배경 |
| `--surface` | 반투명 크림 `rgba(…, ~0.6)` | 카드·패널 면. **반투명**이라 종이 결이 비침(§1.5) |
| `--surface-solid` | 불투명 크림 | 모달·드롭다운·내비 등 오버레이 면(뒤가 비치면 안 되는 곳) |
| `--ink` | `#232418` | 본문·제목 기본 글자색(잉크) |
| `--muted` | `#6f6b57` | 보조 텍스트(메타·설명·placeholder) |
| `--accent` | `#345c4d` | **주 강조**(숲색): 주요 버튼, 링크, 활성 상태, 브랜드 |
| `--accent-ink` | `#f6f3ea` | accent 배경 위 글자색 |
| `--accent-soft` | `#e4ddc9` | accent의 옅은 배경(태그, hover 면, 히어로) |
| `--gold` | `#b98a3c` | **장식 강조**(황동): 별점, 글귀 강조선 등 소수 |
| `--danger` | `#c0392b` | 파괴적/경고: 삭제 버튼, 에러, 좋아요(♥) |
| `--border` | 부드러운 hairline | 경계선·구분선. 종이톤에 가깝게 낮춰 요소가 도드라지지 않게(§1.5) |

**규칙**
- 텍스트는 `--ink`(본문) / `--muted`(보조) 두 단계만. 저대비 남발 금지.
- 강조는 `--accent` 하나를 기본으로. `--gold`는 별점·글귀 같은 "장식적" 강조에만 제한.
- `--danger`는 되돌릴 수 없는 행동과 에러에만. 일반 강조에 빨강을 쓰지 않는다.
- 배경 위계는 `--bg`(바탕) → `--surface`(면) 2단계. 면 위에 또 면을 겹칠 땐 `--border`로 구분.
- 새 상태색(성공 등)이 필요하면 토큰을 추가하고 여기 문서에 등록한 뒤 쓴다.

## 3. Layout — 간격 · 크기 · 배치

### 간격 스케일 (필수: 4의 배수)

모든 여백·패딩·gap은 **4px의 배수**만 사용한다. 어정쩡한 값(9px, 11px, 14px, 18px 등) 금지.

| 스텝 | 값 | 용도 예 |
|------|-----|---------|
| `space-1` | **4px** | 아이콘-텍스트 사이, 미세 간격 |
| `space-2` | **8px** | 관련 요소 사이 기본 gap, 작은 패딩 |
| `space-3` | **12px** | 카드 내부 요소 gap |
| `space-4` | **16px** | 카드 패딩, 컴포넌트 간 기본 간격 |
| `space-5` | **20px** | 넉넉한 카드 패딩 |
| `space-6` | **24px** | 섹션 내부 여백, 페이지 좌우 패딩 |
| `space-8` | **32px** | 히어로 패딩, 섹션 사이 |
| `space-12` | **48px** | 큰 섹션 구분 |
| `space-16` | **64px** | 페이지 하단 여백, 큰 블록 분리 |

### 레이아웃

- 콘텐츠 최대 폭: **960px**(`.container`), 좌우 패딩 `space-6`(24px), 하단 `space-16`(64px).
- 읽기 본문(서평 등)은 한 줄 **약 42em**(≈65자)로 제한해 가독성 유지.
- 형제 요소 배치는 **flex/grid + `gap`**으로. 개별 `margin`으로 간격을 만들지 않는다(상쇄·중복 방지).
- 넓은 콘텐츠(표·코드)는 자체 컨테이너에 `overflow-x: auto`. 페이지 본문은 가로 스크롤 금지.

### 타이포그래피

| 역할 | 폰트 | 크기 | 비고 |
|------|------|------|------|
| Display / 히어로 | `--font-display`(을유1945) | `clamp(1.6rem, 4vw, 2.5rem)` | `text-wrap: balance` |
| 섹션 제목 h2 | `--font-display` | ~1.2rem, 굵게 | 하단 `--border` 실선으로 구분 |
| 소제목 h3 | `--font-display` | ~1.05–1.15rem | |
| 본문 | `--font-body`(을유1945) | 1rem, line-height 1.5 | |
| 읽기 본문(서평) | `--font-body` | 1.12rem, line-height 1.85 | 여유 있는 행간 |
| 보조/메타 | `--font-body` | 0.82rem, `--muted` | `.small` |

- **제목·본문·UI 모두 을유1945**(`--font-display`=`--font-body`). 특정 요소만 다른 폰트로 바꾸지 않는다.
- 숫자가 줄맞춤되는 곳(별점·좋아요 수·페이지·통계)은 `font-variant-numeric: tabular-nums`.
- 말줄임은 `…`(ellipsis) 사용, `...` 금지. 인용부호는 곡선따옴표(“ ”).

## 4. Elevation — 요소가 떠오르는 방식

종이 위에 얇게 얹힌 느낌. 그림자는 **낮고 부드럽게**, 남용하지 않는다. 기본은 그림자 없이 **테두리(`--border`)로 구분**한다.

| 레벨 | 그림자 | 용도 |
|------|--------|------|
| **0 (flat)** | 없음 + `1px solid --border` | 카드·패널·입력의 기본 상태 |
| **1 (raise)** | `0 6px 18px rgba(0,0,0,0.08)` | 카드 hover, 살짝 떠오름 |
| **2 (menu)** | `0 8px 24px rgba(0,0,0,0.12)` | 드롭다운·팝오버·알림 메뉴 |
| **3 (overlay)** | 반투명 배경 `rgba(0,0,0,0.45)` + 면 | 모달. 뒤 콘텐츠를 덮음 |

**규칙**
- 평상시엔 elevation 0(테두리)로 두고, **상호작용(hover)·일시적 표시(메뉴/모달)에만** 그림자를 준다.
- 그림자 색은 항상 중립 검정 알파. 컬러 그림자 금지(단, 다크 배경 위 소수 강조는 예외적 허용).
- 모달은 배경 스크롤 잠금(`overscroll-behavior: contain`)과 함께 쓴다.

---

## 5. Shape — 모양(모서리·선)

살짝 각진 종이/책 느낌. 과한 둥근 모서리를 피한다.

| 요소 | radius |
|------|--------|
| 버튼 | **6px** |
| 카드·패널·모달·히어로 | **8–10px** |
| 입력(input/textarea/select) | **8px** |
| 태그·뱃지·칩·알약형 버튼 | **999px** (완전 알약) |
| 표지 이미지 | 3–4px (거의 각짐) |
| 아바타·아이콘 버튼 | **50%** (원) |

**규칙**
- 컨테이너류는 6~10px의 "살짝 각진" 값으로 통일. 16px 이상 큰 라운드는 쓰지 않는다.
- 알약형(999px)은 태그·칩·상태 뱃지 등 "라벨" 성격에만.
- 구분선은 `1px solid --border`. 히어로·서평 본문 위 강조 구분은 `2px solid`(ink 또는 border).
- 테두리 두께는 1px 기본, 강조 구분만 2px.

## 6. Components — 컴포넌트 규격

공통 요소는 아래 규격을 재사용한다. 새 변형이 필요하면 boolean prop을 늘리지 말고 **명시적 컴포넌트/클래스**로 분리한다.

### 버튼 (`.btn`)
- 기본: 배경 `--accent`, 글자 `--accent-ink`, radius 6px, 패딩 `space-2`/`space-4`(8/16), 굵기 500.
- `.btn.ghost`: 투명 배경 + `--border` 테두리 + `--accent` 글자. 보조 액션.
- `.btn.small`: 패딩 축소(6/12), 0.85rem. `.btn.full`: 폭 100%. `.btn.danger`: 배경 `--danger`.
- 아이콘 전용 버튼은 **반드시 `aria-label`**. 이모지는 `aria-hidden`.

### 카드
- 컨테이너: `--surface` 배경 + `1px solid --border` + radius 8px, 내부 패딩 `space-4`(16).
- hover 시 elevation 1.
- **서평 카드(`.review-card`)**: 서평 내용이 주인공 — 표지는 작게, 본문(`.sq-review`)은 `--ink`로 최대 6줄, 하트·작성자는 하단 footer로 분리.

### 태그·뱃지 (`.tag`, `.page-badge`)
- 알약형(999px), 작은 글씨(0.72–0.78rem), `--accent-soft` 배경 + `--accent` 글자.

### 입력 (`input`, `textarea`, `select`)
- `#fff` 배경 + `--border` + radius 8px, 패딩 `space-2`~(8/12).
- 포커스: `:focus-visible`에 `2px solid --accent-soft` outline + `--accent` 테두리.
- 라벨은 `<label>` 연결 또는 `aria-label`. 아이디/이메일류는 `spellCheck={false}`, 적절한 `autoComplete`.

### 별점 (`StarRatingDisplay` / `StarRatingInput`)
- 표시용과 입력용을 **분리**한다. 표시용은 `role="img"` + aria-label(비버튼), 입력용은 별마다 버튼.
- 채운 별 `--gold`, 빈 별 회색.

### 모달 (`Modal`)
- elevation 3(반투명 배경 + `--surface` 면), radius 10px.
- `role="dialog"` `aria-modal` `aria-labelledby`, **Escape 닫기**, 열릴 때 포커스 이동, 배경 스크롤 잠금.

### 내비게이션 / 탭 / 칩
- 활성 상태는 `--accent`(글자/밑줄/배경)로 표시. 비활성은 `--muted`.
- 알림·아바타 등 토글 버튼은 `aria-haspopup`/`aria-expanded`.

### 접근성 공통
- 모든 상호작용 요소는 `:focus-visible` 포커스 링 필수.
- 비동기 에러/성공 메시지는 `role="alert"`/`role="status"`.
- 애니메이션은 `prefers-reduced-motion: reduce`를 존중.
- 이미지는 `width`/`height` 지정(CLS 방지), 접힘 아래는 `loading="lazy"`.

- [ ] 모든 간격이 **4의 배수**(§3 스케일)인가
- [ ] 제목·본문 모두 을유1945, 숫자=tabular-nums 인가
- [ ] 종이 질감 배경·요소 반투명·부드러운 경계(§1.5)를 지켰는가
- [ ] 그림자는 hover/메뉴/모달에만(§4), 평상시엔 테두리인가
- [ ] 모서리 6~10px(라벨만 999px), 큰 라운드 안 썼는가
- [ ] 컴포넌트는 §6 규격을 재사용했는가(임의 스타일 X)
- [ ] 포커스 링·aria·이미지 크기 등 접근성 충족했는가
